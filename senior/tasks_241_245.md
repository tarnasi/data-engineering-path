## Task 241: Real-Time Analytics Architecture — Sub-Minute Decision Making

### Business Context
NovaMart's logistics VP comes to you: "I need to optimize delivery routing in real-time. Today, routes are planned at 6 AM using yesterday's data. By noon, 40% of routes are suboptimal because of traffic, weather, and order cancellations. Amazon adjusts routes every 15 minutes. We need data that's less than 1 minute old." This requires a fundamental shift: from batch-first (data arrives in Snowflake daily) to a real-time analytics layer that can power operational decisions.

### Objective
Build a real-time analytics architecture that serves sub-minute-fresh data for operational decision-making, while maintaining the batch layer for historical analysis.

### Technical Requirements
- **Tools**: Apache Kafka, Apache Flink, Druid/Pinot (real-time OLAP), Python, SQL
- **Skills**: Lambda/Kappa architecture, real-time OLAP, stream processing, materialized views

   # realtime/flink_aggregations.py
   """
   Flink SQL job: real-time order and logistics metrics.
   Aggregates tumbling windows for sub-minute analytics.
   """

   FLINK_SQL_JOBS = {
       "realtime_order_metrics": """
           -- Real-time order metrics (1-minute tumbling window)
           CREATE TABLE order_events (
               order_id STRING,
               customer_id STRING,
               total_amount DECIMAL(12,2),
               region STRING,
               channel STRING,
               status STRING,
               event_time TIMESTAMP(3),
               WATERMARK FOR event_time AS event_time - INTERVAL '10' SECOND
           ) WITH (
               'connector' = 'kafka',
               'topic' = 'order-events',
               'properties.bootstrap.servers' = 'kafka:9092',
               'format' = 'json',
               'scan.startup.mode' = 'latest-offset'
           );

           CREATE TABLE realtime_order_metrics (
               window_start TIMESTAMP(3),
               window_end TIMESTAMP(3),
               region STRING,
               channel STRING,
               order_count BIGINT,
               total_revenue DECIMAL(18,2),
               avg_order_value DECIMAL(12,2),
               PRIMARY KEY (window_start, region, channel) NOT ENFORCED
           ) WITH (
               'connector' = 'upsert-kafka',
               'topic' = 'realtime-order-metrics',
               'properties.bootstrap.servers' = 'kafka:9092',
               'key.format' = 'json',
               'value.format' = 'json'
           );

           INSERT INTO realtime_order_metrics
           SELECT
               TUMBLE_START(event_time, INTERVAL '1' MINUTE) AS window_start,
               TUMBLE_END(event_time, INTERVAL '1' MINUTE) AS window_end,
               region,
               channel,
               COUNT(*) AS order_count,
               SUM(total_amount) AS total_revenue,
               AVG(total_amount) AS avg_order_value
           FROM order_events
           WHERE status != 'cancelled'
           GROUP BY
               TUMBLE(event_time, INTERVAL '1' MINUTE),
               region,
               channel;
       """,

       "realtime_logistics_metrics": """
           -- Real-time delivery tracking (30-second tumbling window)
           CREATE TABLE shipment_events (
               shipment_id STRING,
               order_id STRING,
               driver_id STRING,
               status STRING,
               latitude DOUBLE,
               longitude DOUBLE,
               estimated_arrival TIMESTAMP(3),
               event_time TIMESTAMP(3),
               WATERMARK FOR event_time AS event_time - INTERVAL '5' SECOND
           ) WITH (
               'connector' = 'kafka',
               'topic' = 'shipment-events',
               'properties.bootstrap.servers' = 'kafka:9092',
               'format' = 'json'
           );

           CREATE TABLE realtime_logistics (
               window_start TIMESTAMP(3),
               region STRING,
               active_deliveries BIGINT,
               avg_eta_minutes DOUBLE,
               delayed_deliveries BIGINT,
               on_time_rate DOUBLE,
               PRIMARY KEY (window_start, region) NOT ENFORCED
           ) WITH (
               'connector' = 'upsert-kafka',
               'topic' = 'realtime-logistics-metrics',
               'properties.bootstrap.servers' = 'kafka:9092',
               'key.format' = 'json',
               'value.format' = 'json'
           );

           INSERT INTO realtime_logistics
           SELECT
               TUMBLE_START(event_time, INTERVAL '30' SECOND),
               region,
               COUNT(DISTINCT shipment_id),
               AVG(TIMESTAMPDIFF(MINUTE, event_time, estimated_arrival)),
               COUNT(CASE WHEN status = 'delayed' THEN 1 END),
               CAST(COUNT(CASE WHEN status = 'on_time' THEN 1 END) AS DOUBLE) 
                   / NULLIF(COUNT(*), 0)
           FROM shipment_events
           WHERE status IN ('in_transit', 'delayed', 'on_time')
           GROUP BY
               TUMBLE(event_time, INTERVAL '30' SECOND),
               region;
       """,
   }

4. **Build the query router**:
   ```python
   # realtime/query_router.py
   class QueryRouter:
       """Route queries to Druid (real-time) or Snowflake (historical)."""

       def __init__(self, druid_client, snowflake_conn):
           self.druid = druid_client
           self.snowflake = snowflake_conn

       def query(self, metric: str, time_range: str,
                 dimensions: list[str] = None) -> dict:
           """
           Automatically route based on time range:
           - Last 24 hours → Druid (sub-second response, sub-minute fresh)
           - Older than 24 hours → Snowflake (full history, complex analytics)
           """
           if self._is_recent(time_range):
               return self._query_druid(metric, time_range, dimensions)
           else:
               return self._query_snowflake(metric, time_range, dimensions)

       def _is_recent(self, time_range: str) -> bool:
           return time_range in ("last_1h", "last_6h", "last_24h", "today")

       def _query_druid(self, metric, time_range, dimensions):
           query = {
               "queryType": "groupBy",
               "dataSource": f"realtime_order_metrics",
               "intervals": [self._time_range_to_interval(time_range)],
               "granularity": "minute",
               "dimensions": dimensions or [],
               "aggregations": [
                   {"type": "longSum", "name": metric, "fieldName": metric}
               ],
           }
           result = self.druid.query(query)
           return {"source": "druid", "freshness": "<1min", "data": result}

       def _query_snowflake(self, metric, time_range, dimensions):
           # Build SQL against Snowflake semantic layer
           sql = f"SELECT ... FROM gold.fact_orders WHERE ..."
           result = self.snowflake.cursor().execute(sql).fetchall()
           return {"source": "snowflake", "freshness": "<1hour", "data": result}

       def _time_range_to_interval(self, time_range: str) -> str:
           from datetime import datetime, timedelta
           now = datetime.utcnow()
           ranges = {
               "last_1h": timedelta(hours=1),
               "last_6h": timedelta(hours=6),
               "last_24h": timedelta(hours=24),
           }
           delta = ranges.get(time_range, timedelta(hours=24))
           start = (now - delta).strftime("%Y-%m-%dT%H:%M:%S")
           end = now.strftime("%Y-%m-%dT%H:%M:%S")
           return f"{start}/{end}"
   ```

       comment = string

     name            = "${lower(var.database_name)}_monitor"

     frequency       = "MONTHLY"
     start_timestamp = "IMMEDIATELY"

     notify_triggers            = [75, 90]
     suspend_triggers           = [100]
     suspend_immediate_triggers = [150]
   }

   output "database_name" {
     value = snowflake_database.this.name
   }

   output "schema_names" {
     value = [for s in snowflake_schema.schemas : s.name]
   }

1. **Design the graph model for fraud detection**:
   ```python
   # data_models/graph/fraud_graph.py
   import networkx as nx
   from collections import defaultdict

   class FraudGraph:
       """
       Graph model for detecting fraud rings.
       Nodes = customers
       Edges = shared attributes (address, device, card, IP)
       """

       def __init__(self):
           self.graph = nx.Graph()  # Undirected: shared relationships

       def build_from_data(self, customer_data: list[dict]):
           """Build graph from customer records."""
           # Index shared attributes
           by_address = defaultdict(set)
           by_device = defaultdict(set)
           by_card = defaultdict(set)
           by_ip = defaultdict(set)

           for c in customer_data:
               cid = c["customer_id"]
               self.graph.add_node(cid, **{
                   "name": c.get("name", ""),
                   "account_age_days": c.get("account_age_days", 0),
                   "total_orders": c.get("total_orders", 0),
                   "total_chargebacks": c.get("total_chargebacks", 0),
               })

               if c.get("address_hash"):
                   by_address[c["address_hash"]].add(cid)
               if c.get("device_fingerprint"):
                   by_device[c["device_fingerprint"]].add(cid)
               if c.get("card_hash"):
                   by_card[c["card_hash"]].add(cid)
               if c.get("ip_address"):
                   by_ip[c["ip_address"]].add(cid)

           # Create edges for shared attributes
           for attr_name, groups in [
               ("shared_address", by_address),
               ("shared_device", by_device),
               ("shared_card", by_card),
               ("shared_ip", by_ip),
           ]:
               for shared_val, customers in groups.items():
                   if len(customers) > 1:
                       customer_list = list(customers)
                       for i in range(len(customer_list)):
                           for j in range(i + 1, len(customer_list)):
                               if self.graph.has_edge(customer_list[i],
                                                       customer_list[j]):
                                   # Add relationship type to existing edge
                                   existing = self.graph.edges[
                                       customer_list[i], customer_list[j]
                                   ]
                                   existing["shared_attributes"].append(attr_name)
                                   existing["risk_score"] += 1
                               else:
                                   self.graph.add_edge(
                                       customer_list[i],
                                       customer_list[j],
                                       shared_attributes=[attr_name],
                                       risk_score=1,
                                   )

       def detect_fraud_rings(self, min_size: int = 3,
                              min_risk: int = 2) -> list[dict]:
           """Detect clusters of connected customers (potential fraud rings)."""
           rings = []
           for component in nx.connected_components(self.graph):
               if len(component) < min_size:
                   continue

               subgraph = self.graph.subgraph(component)
               max_risk = max(
                   d.get("risk_score", 0)
                   for _, _, d in subgraph.edges(data=True)
               )
               if max_risk < min_risk:
                   continue

               # Calculate ring risk score
               total_chargebacks = sum(
                   self.graph.nodes[n].get("total_chargebacks", 0)
                   for n in component
               )
               avg_account_age = sum(
                   self.graph.nodes[n].get("account_age_days", 0)
                   for n in component
               ) / len(component)

               rings.append({
                   "ring_id": f"FR-{hash(frozenset(component)) % 10000:04d}",
                   "size": len(component),
                   "customers": list(component),
                   "max_edge_risk": max_risk,
                   "total_chargebacks": total_chargebacks,
                   "avg_account_age_days": avg_account_age,
                   "shared_attributes": self._ring_shared_attributes(subgraph),
                   "risk_assessment": (
                       "HIGH" if total_chargebacks > 0 or avg_account_age < 30
                       else "MEDIUM"
                   ),
               })

           return sorted(rings, key=lambda r: r["size"], reverse=True)

       def _ring_shared_attributes(self, subgraph) -> dict:
           attrs = defaultdict(int)
           for _, _, data in subgraph.edges(data=True):
               for attr in data.get("shared_attributes", []):
                   attrs[attr] += 1
           return dict(attrs)

       def customer_risk_score(self, customer_id: str) -> dict:
           """Calculate fraud risk for a single customer."""
           if customer_id not in self.graph:
               return {"customer_id": customer_id, "risk": "unknown"}

           neighbors = list(self.graph.neighbors(customer_id))
           neighbor_chargebacks = sum(
               self.graph.nodes[n].get("total_chargebacks", 0)
               for n in neighbors
           )

           # Degree centrality = how connected
           centrality = nx.degree_centrality(self.graph).get(customer_id, 0)

           risk_factors = []
           if len(neighbors) > 5:
               risk_factors.append(f"Connected to {len(neighbors)} accounts")
           if neighbor_chargebacks > 0:
               risk_factors.append(
                   f"Network has {neighbor_chargebacks} chargebacks"
               )
           if centrality > 0.1:
               risk_factors.append(f"High centrality: {centrality:.3f}")

           return {
               "customer_id": customer_id,
               "connections": len(neighbors),
               "centrality": round(centrality, 4),
               "neighbor_chargebacks": neighbor_chargebacks,
               "risk_factors": risk_factors,
               "risk_level": (
                   "HIGH" if neighbor_chargebacks > 2 or centrality > 0.1
                   else "MEDIUM" if neighbors
                   else "LOW"
               ),
           }
   ```

2. **Design time-series model for IoT sensor data**:

   -- data_models/timeseries/warehouse_sensors.sql

   -- Optimized time-series table with Snowflake clustering
   CREATE TABLE IF NOT EXISTS IOT.WAREHOUSE_SENSOR_READINGS (
       reading_id      NUMBER AUTOINCREMENT,
       sensor_id       VARCHAR(50) NOT NULL,
       warehouse_id    VARCHAR(20) NOT NULL,
       zone            VARCHAR(50) NOT NULL,
       metric_type     VARCHAR(30) NOT NULL,  -- temperature, humidity, vibration
       value           FLOAT NOT NULL,
       unit            VARCHAR(10),
       reading_time    TIMESTAMP_NTZ NOT NULL,
       ingested_at     TIMESTAMP_NTZ DEFAULT CURRENT_TIMESTAMP()
   )
   CLUSTER BY (reading_time, warehouse_id, sensor_id);

   -- Pre-aggregated rollups for fast dashboarding
   CREATE TABLE IF NOT EXISTS IOT.SENSOR_ROLLUP_1MIN AS
   SELECT
       DATE_TRUNC('minute', reading_time) AS minute,
       sensor_id,
       warehouse_id,
       zone,
       metric_type,
       AVG(value) AS avg_value,
       MIN(value) AS min_value,
       MAX(value) AS max_value,
       STDDEV(value) AS stddev_value,
       COUNT(*) AS reading_count
   FROM IOT.WAREHOUSE_SENSOR_READINGS
   GROUP BY 1, 2, 3, 4, 5;

   -- Anomaly detection using statistical methods
   WITH sensor_stats AS (
       SELECT
           sensor_id,
           metric_type,
           AVG(value) AS mean_value,
           STDDEV(value) AS stddev_value
       FROM IOT.WAREHOUSE_SENSOR_READINGS
       WHERE reading_time >= DATEADD(day, -7, CURRENT_TIMESTAMP())
       GROUP BY sensor_id, metric_type
   ),
   readings_with_z AS (
       SELECT
           r.*,
           s.mean_value,
           s.stddev_value,
           ABS(r.value - s.mean_value) / NULLIF(s.stddev_value, 0) AS z_score
       FROM IOT.WAREHOUSE_SENSOR_READINGS r
       JOIN sensor_stats s
           ON r.sensor_id = s.sensor_id AND r.metric_type = s.metric_type
       WHERE r.reading_time >= DATEADD(hour, -24, CURRENT_TIMESTAMP())
   )
   SELECT
       reading_time,
       sensor_id,
       warehouse_id,
       zone,
       metric_type,
       value,
       mean_value,
       z_score,
       CASE
           WHEN z_score > 3 THEN 'CRITICAL'
           WHEN z_score > 2 THEN 'WARNING'
           ELSE 'NORMAL'
       END AS anomaly_level
   FROM readings_with_z
   WHERE z_score > 2
   ORDER BY z_score DESC;
   ```

3. **Build time-series analysis tools**:
   ```python
   # data_models/timeseries/analysis.py
   from dataclasses import dataclass
   from typing import Optional

   @dataclass
   class TimeSeriesAnalysis:
       sensor_id: str
       metric_type: str
       period: str
       mean: float
       stddev: float
       min_val: float
       max_val: float
       anomaly_count: int
       trend: str  # "increasing", "decreasing", "stable"

   class SensorAnalyzer:
       def __init__(self, snowflake_conn):
           self.conn = snowflake_conn

       def analyze_sensor(self, sensor_id: str, metric_type: str,
                          hours: int = 24) -> TimeSeriesAnalysis:
           """Analyze a sensor's recent readings."""
           result = self.conn.cursor().execute(f"""
               WITH readings AS (
                   SELECT value, reading_time,
                          ROW_NUMBER() OVER (ORDER BY reading_time) AS rn,
                          COUNT(*) OVER () AS total
                   FROM IOT.WAREHOUSE_SENSOR_READINGS
                   WHERE sensor_id = '{sensor_id}'
                       AND metric_type = '{metric_type}'
                       AND reading_time >= DATEADD(hour, -{hours}, CURRENT_TIMESTAMP())
               ),
               stats AS (
                   SELECT
                       AVG(value) AS mean_val,
                       STDDEV(value) AS std_val,
                       MIN(value) AS min_val,
                       MAX(value) AS max_val,
                       COUNT(CASE WHEN ABS(value - AVG(value) OVER ()) / 
                           NULLIF(STDDEV(value) OVER (), 0) > 2 THEN 1 END) AS anomalies
                   FROM readings
               ),
               trend AS (
                   SELECT
                       REGR_SLOPE(value, rn) AS slope
                   FROM readings
               )
               SELECT s.*, t.slope FROM stats s, trend t
           """).fetchone()

           trend = "stable"
           if result and result[-1]:
               if result[-1] > 0.01:
                   trend = "increasing"
               elif result[-1] < -0.01:
                   trend = "decreasing"

           return TimeSeriesAnalysis(
               sensor_id=sensor_id,
               metric_type=metric_type,
               period=f"last_{hours}h",
               mean=result[0] if result else 0,
               stddev=result[1] if result else 0,
               min_val=result[2] if result else 0,
               max_val=result[3] if result else 0,
               anomaly_count=result[4] if result else 0,
               trend=trend,
           )

       def detect_correlated_anomalies(self, warehouse_id: str,
                                        hours: int = 24) -> list[dict]:
           """Find sensors that have anomalies at the same time."""
           results = self.conn.cursor().execute(f"""
               WITH anomalies AS (
                   SELECT
                       sensor_id, reading_time, value, metric_type,
                       ABS(value - AVG(value) OVER (
                           PARTITION BY sensor_id
                           ORDER BY reading_time
                           ROWS BETWEEN 60 PRECEDING AND CURRENT ROW
                       )) / NULLIF(STDDEV(value) OVER (
                           PARTITION BY sensor_id
                           ORDER BY reading_time
                           ROWS BETWEEN 60 PRECEDING AND CURRENT ROW
                       ), 0) AS z_score
                   FROM IOT.WAREHOUSE_SENSOR_READINGS
                   WHERE warehouse_id = '{warehouse_id}'
                       AND reading_time >= DATEADD(hour, -{hours}, CURRENT_TIMESTAMP())
               )
               SELECT
                   DATE_TRUNC('minute', reading_time) AS anomaly_minute,
                   COUNT(DISTINCT sensor_id) AS sensors_affected,
                   LISTAGG(DISTINCT sensor_id, ', ') AS sensor_list
               FROM anomalies
               WHERE z_score > 3
               GROUP BY 1
               HAVING COUNT(DISTINCT sensor_id) > 1
               ORDER BY sensors_affected DESC
           """).fetchall()

           return [
               {
                   "time": str(r[0]),
                   "sensors_affected": r[1],
                   "sensors": r[2],
               }
               for r in results
           ]
   ```

   # How We Reduced Data Incidents by 90% in 18 Months

   *Published on the NovaMart Engineering Blog*

   ---

   At NovaMart, "the data is wrong" used to be the most common Slack message 
   in #data-questions. Our weekly data incident count peaked at 12. Dashboards 
   contradicted each other. Finance couldn't trust the numbers.

   18 months later, our weekly incident count is consistently below 1, our 
   data freshness SLA is 99.5%, and the CFO presents board numbers generated 
   directly from our data platform. Here's how we got there.

   ## The Problem: Death by a Thousand Paper Cuts

   We tracked every data quality issue for 30 days. The breakdown:

   | Category | % of Issues | Example |
   |----------|:-----------:|---------|
   | Stale data | 35% | Dashboard showing yesterday's numbers at 2 PM |
   | Metric inconsistency | 25% | Two dashboards, two definitions of "revenue" |
   | Schema changes | 20% | Source DB column renamed, pipeline crashed |
   | Data loss | 10% | CDC pipeline missed deletions |
   | Duplicates | 10% | Flink exactly-once wasn't configured correctly |

   ## Level 1: Automated Testing (Months 1-3)
   
   We added dbt tests to every model. Not just `not_null` and `unique` — 
   business logic tests:

   ```sql
   -- Revenue can't be negative
   SELECT * FROM {{ ref('fact_orders') }} 
   WHERE total_amount_usd < 0

   -- Order date must be within the last 2 years
   SELECT * FROM {{ ref('fact_orders') }}
   WHERE order_date < DATEADD(year, -2, CURRENT_DATE())
       OR order_date > CURRENT_DATE()
   ```

   **Result**: 30% reduction in incidents.

   ## Level 2: Data Contracts (Months 4-9)

   Testing catches problems AFTER they happen. We needed prevention.

   We implemented data contracts between producers and consumers. Every 
   upstream schema change now requires:
   1. A PR with the schema change
   2. Automated impact analysis (who's downstream?)
   3. Notification to affected teams
   4. 48-hour window before breaking changes go live

   **Result**: Schema-change incidents dropped to zero.

   ## Level 3: Semantic Layer (Months 10-15)

   We unified every business metric into a single semantic layer. "Revenue" 
   went from 7 definitions to 1. Every dashboard, API, and notebook pulls 
   from the same metric definition.

   **Result**: Metric inconsistency incidents dropped to zero.

   ## Level 4: SLAs and Culture (Months 15-18)

   Technology wasn't enough. We needed cultural change:
   - Every dataset has an owner
   - Every dataset has an SLA
   - Quality scores are visible to everyone
   - Teams own their quality (not the platform team)

   **Result**: Remaining incidents dropped 60%.

   1. **Anomaly detection ML**: Too many false positives. Simple z-score thresholds

   2. **Data quality SLA penalties**: Punishing teams for bad data created blame 
      culture. Switched to celebration of quality improvements.
   3. **100% test coverage**: Diminishing returns past 85%. Focus on the critical

### Common Mistakes
- Too much jargon — conference audiences are diverse; explain terms the first time you use them
- Self-promotion over storytelling — "We're awesome" is boring; "We failed, then fixed it" is compelling
- Company-specific without takeaways — abstract the lessons; the audience doesn't care about your Kafka topics, they care about the decision framework
- Not submitting enough CFPs — most conferences have 5-10% acceptance rate; submit to 10+ conferences
