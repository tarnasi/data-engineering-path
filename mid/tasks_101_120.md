## Task 101: Cloud Data Warehouse — Setup and Migration Planning

### Business Context
NovaMart's PostgreSQL-based warehouse is hitting limits: slow analytical queries (30+ minutes for complex reports), storage costs growing linearly, and engineering overhead maintaining indexes and vacuuming. Leadership approved the migration to Snowflake (or BigQuery). Ravi says: "Plan the migration. We're not doing a big-bang cutover — we need a phased approach where both systems run in parallel."

### Objective
Design the cloud data warehouse architecture, set up the Snowflake (or BigQuery) environment, and build a migration plan that moves NovaMart from PostgreSQL to the cloud warehouse without downtime.

### Technical Requirements
- **Tools**: Snowflake or BigQuery, SQL, Terraform (optional), dbt
- **Skills**: Cloud warehouse architecture, virtual warehouses, resource sizing, migration planning

### Step-by-Step Guidance
1. **Set up the Snowflake environment** (or BigQuery project):
   ```sql
   -- Snowflake setup
   CREATE DATABASE novamart_dw;

   CREATE SCHEMA novamart_dw.raw;        -- landing zone for raw data
   CREATE SCHEMA novamart_dw.staging;     -- transformed staging layer
   CREATE SCHEMA novamart_dw.analytics;   -- mart layer for consumers
   CREATE SCHEMA novamart_dw.scratch;     -- ad-hoc analyst workspace

   -- Separate warehouses for different workloads
   CREATE WAREHOUSE etl_wh
       WITH WAREHOUSE_SIZE = 'MEDIUM'
       AUTO_SUSPEND = 300
       AUTO_RESUME = TRUE
       INITIALLY_SUSPENDED = TRUE;

   CREATE WAREHOUSE analytics_wh
       WITH WAREHOUSE_SIZE = 'SMALL'
       AUTO_SUSPEND = 60
       AUTO_RESUME = TRUE;

   CREATE WAREHOUSE dbt_wh
       WITH WAREHOUSE_SIZE = 'SMALL'
       AUTO_SUSPEND = 120
       AUTO_RESUME = TRUE;
   ```

2. **Design the roles and access control**:
   ```sql
   CREATE ROLE data_engineer;
   CREATE ROLE data_analyst;
   CREATE ROLE data_scientist;
   CREATE ROLE dbt_runner;

   GRANT USAGE ON WAREHOUSE etl_wh TO ROLE data_engineer;
   GRANT USAGE ON WAREHOUSE analytics_wh TO ROLE data_analyst;
   GRANT USAGE ON WAREHOUSE dbt_wh TO ROLE dbt_runner;

   GRANT ALL ON SCHEMA raw TO ROLE data_engineer;
   GRANT SELECT ON ALL TABLES IN SCHEMA analytics TO ROLE data_analyst;
   ```

3. **Write the migration plan**:
   ```markdown
   ## Migration Phases

   ### Phase 1 — Dual Write (Weeks 1-2)
   - Continue writing to PostgreSQL (primary)
   - Replicate raw data to Snowflake (read-only copy)
   - dbt runs against both, compare outputs

   ### Phase 2 — Shadow Mode (Weeks 3-4)
   - Snowflake becomes the primary for dbt/analytics
   - PostgreSQL still receives writes
   - Compare query results: Snowflake vs PostgreSQL

   ### Phase 3 — Cutover (Week 5)
   - Snowflake becomes primary for all reads and writes
   - PostgreSQL becomes backup (30-day retention)
   - Redirect all dashboards and consumers

   ### Phase 4 — Decommission (Weeks 6-8)
   - After 30 days with no issues, archive PostgreSQL
   - Remove dual-write code paths
   ```

4. **Benchmark**: run the 10 slowest PostgreSQL queries on Snowflake, compare execution times
5. **Data loading test**: load 1 day of production data into Snowflake, verify row counts match

### Expected Output
- Setup scripts: `tasks/101_cloud_warehouse/setup/`
- Migration plan: `tasks/101_cloud_warehouse/migration_plan.md`
- Benchmark results: `tasks/101_cloud_warehouse/benchmark.md`
- Role/access design: `tasks/101_cloud_warehouse/access_control.sql`

### Common Mistakes
- Big-bang migration — too risky; always run in parallel first
- Same warehouse for ETL and analytics — ETL monopolizes compute, analysts can't query
- Not testing with production-volume data — a test with 100 rows doesn't reveal performance issues
- Ignoring Snowflake's pricing model — compute charges per second, storage per TB/month

## Task 102: Cloud Data Loading — External Stages and COPY

### Business Context
NovaMart's raw data lives in S3 (or GCS). You need to load it into Snowflake efficiently. The initial load is 500GB of historical data. Daily incremental loads are 2-5GB. Ravi says: "Use external stages and COPY commands — not INSERT statements. And set up Snowpipe for continuous loading."

### Objective
Set up external stages, bulk-load historical data using COPY INTO, and configure Snowpipe for continuous near-real-time loading from S3/GCS.

### Technical Requirements
- **Tools**: Snowflake (Stages, COPY, Snowpipe) or BigQuery (External Tables, Load Jobs), AWS S3 or GCS
- **Skills**: Cloud storage integration, bulk loading, continuous ingestion, file formats

### Step-by-Step Guidance
1. **Create an external stage**:
   ```sql
   -- Snowflake: S3 external stage
   CREATE OR REPLACE STAGE novamart_s3_stage
       URL = 's3://novamart-data-lake/raw/'
       STORAGE_INTEGRATION = novamart_s3_integration
       FILE_FORMAT = (
           TYPE = 'PARQUET'
       );

   -- List files in the stage
   LIST @novamart_s3_stage/orders/;
   ```

2. **Bulk load historical data**:
   ```sql
   -- Load all Parquet files from S3 into Snowflake
   COPY INTO raw.orders
   FROM @novamart_s3_stage/orders/
   FILE_FORMAT = (TYPE = 'PARQUET')
   MATCH_BY_COLUMN_NAME = CASE_INSENSITIVE
   ON_ERROR = 'CONTINUE';  -- don't fail the whole load for one bad file

   -- Check load history
   SELECT * FROM TABLE(INFORMATION_SCHEMA.COPY_HISTORY(
       TABLE_NAME => 'raw.orders',
       START_TIME => DATEADD(hours, -24, CURRENT_TIMESTAMP())
   ));
   ```

3. **Set up file format for CSVs**:
   ```sql
   CREATE OR REPLACE FILE FORMAT csv_format
       TYPE = 'CSV'
       FIELD_DELIMITER = ','
       SKIP_HEADER = 1
       NULL_IF = ('NULL', 'null', '')
       EMPTY_FIELD_AS_NULL = TRUE
       ERROR_ON_COLUMN_COUNT_MISMATCH = FALSE
       TRIM_SPACE = TRUE;
   ```

4. **Configure Snowpipe for continuous loading**:
   ```sql
   CREATE OR REPLACE PIPE novamart_orders_pipe
       AUTO_INGEST = TRUE
   AS
   COPY INTO raw.orders
   FROM @novamart_s3_stage/orders/incremental/
   FILE_FORMAT = (TYPE = 'PARQUET')
   MATCH_BY_COLUMN_NAME = CASE_INSENSITIVE;

   -- Check pipe status
   SELECT SYSTEM$PIPE_STATUS('novamart_orders_pipe');
   ```

5. **Handle schema evolution in Parquet files**:
   ```sql
   -- Variant column approach: load everything as semi-structured
   CREATE TABLE raw.orders_variant (
       raw_data VARIANT,
       filename VARCHAR DEFAULT METADATA$FILENAME,
       loaded_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP()
   );

   -- Query semi-structured data
   SELECT
       raw_data:order_id::INTEGER AS order_id,
       raw_data:user_id::INTEGER AS user_id,
       raw_data:total_amount::DECIMAL(10,2) AS total_amount,
       raw_data:order_date::DATE AS order_date
   FROM raw.orders_variant;
   ```

6. **Performance optimization**:
   - Use `CLUSTER BY` for large tables
   - Partition landing zone by date in S3: `s3://bucket/orders/dt=2024-07-20/`
   - Use `SIZE_LIMIT` to control file sizes for Snowpipe

### Expected Output
- Stage setup: `tasks/102_cloud_loading/stages.sql`
- Bulk load scripts: `tasks/102_cloud_loading/bulk_load.sql`
- Snowpipe config: `tasks/102_cloud_loading/snowpipe.sql`
- Loading performance report with timing for different file formats and sizes

### Common Mistakes
- INSERT INTO from SELECT on external stage — orders of magnitude slower than COPY
- Not partitioning S3 data by date — full scans on every load
- Snowpipe with very small files — overhead per file makes it inefficient; batch into 100-250MB files
- Ignoring COPY_HISTORY — can't debug load failures without checking the log

## Task 104: Advanced Airflow — Custom Operators and Plugins

### Business Context
NovaMart's Airflow instance has growing copy-paste: every DAG that loads to Snowflake has the same 30 lines of connection setup, query execution, and error handling. Every quality check repeats the same pattern. Ravi says: "Build custom operators. Our DAG code should express *what* to do, not *how* to do it."

### Objective
Build custom Airflow operators for NovaMart's common patterns: Snowflake loading, quality checking, SLA tracking, and alerting. Package them as an Airflow plugin.

### Technical Requirements
- **Tools**: Airflow (Operator API, Hooks, Plugins)
- **Skills**: Custom operators, hooks, plugins, operator composition, testing operators

### Step-by-Step Guidance
1. **Build a Snowflake load operator**:
   ```python
   from airflow.models import BaseOperator
   from airflow.providers.snowflake.hooks.snowflake import SnowflakeHook

   class SnowflakeCopyOperator(BaseOperator):
       template_fields = ('stage_path', 'target_table', 'file_format')

       def __init__(self,
                    snowflake_conn_id: str,
                    stage_path: str,
                    target_table: str,
                    file_format: str = 'PARQUET',
                    on_error: str = 'CONTINUE',
                    warehouse: str = 'etl_wh',
                    **kwargs):
           super().__init__(**kwargs)
           self.snowflake_conn_id = snowflake_conn_id
           self.stage_path = stage_path
           self.target_table = target_table
           self.file_format = file_format
           self.on_error = on_error
           self.warehouse = warehouse

       def execute(self, context):
           hook = SnowflakeHook(snowflake_conn_id=self.snowflake_conn_id)
           hook.run(f"USE WAREHOUSE {self.warehouse}")

           sql = f"""
           COPY INTO {self.target_table}
           FROM @{self.stage_path}
           FILE_FORMAT = (TYPE = '{self.file_format}')
           ON_ERROR = '{self.on_error}'
           """
           result = hook.run(sql)

           # Push row count to XCom for downstream tasks
           count_result = hook.get_first(f"SELECT COUNT(*) FROM {self.target_table}")
           context['ti'].xcom_push(key='rows_loaded', value=count_result[0])
           self.log.info(f"Loaded {count_result[0]} rows into {self.target_table}")
   ```

2. **Build a data quality operator**:
   ```python
   class DataQualityOperator(BaseOperator):
       template_fields = ('table', 'checks')

       def __init__(self,
                    snowflake_conn_id: str,
                    table: str,
                    checks: list,
                    fail_on_error: bool = True,
                    **kwargs):
           super().__init__(**kwargs)
           self.snowflake_conn_id = snowflake_conn_id
           self.table = table
           self.checks = checks
           self.fail_on_error = fail_on_error

       def execute(self, context):
           hook = SnowflakeHook(snowflake_conn_id=self.snowflake_conn_id)
           failures = []

           for check in self.checks:
               if check['type'] == 'row_count_min':
                   count = hook.get_first(f"SELECT COUNT(*) FROM {self.table}")[0]
                   if count < check['threshold']:
                       failures.append(f"Row count {count} < {check['threshold']}")

               elif check['type'] == 'null_check':
                   for col in check['columns']:
                       null_count = hook.get_first(
                           f"SELECT COUNT(*) FROM {self.table} WHERE {col} IS NULL"
                       )[0]
                       total = hook.get_first(f"SELECT COUNT(*) FROM {self.table}")[0]
                       null_rate = null_count / max(total, 1)
                       if null_rate > check.get('threshold', 0.05):
                           failures.append(f"{col}: null rate {null_rate:.2%}")

               elif check['type'] == 'freshness':
                   max_ts = hook.get_first(
                       f"SELECT MAX({check['column']}) FROM {self.table}"
                   )[0]
                   age_hours = (datetime.utcnow() - max_ts).total_seconds() / 3600
                   if age_hours > check['max_hours']:
                       failures.append(f"Data is {age_hours:.1f}h old (max: {check['max_hours']}h)")

           if failures:
               msg = f"Quality checks failed for {self.table}: {failures}"
               if self.fail_on_error:
                   raise AirflowException(msg)
               else:
                   self.log.warning(msg)
   ```

3. **Build an SLA tracking operator**:
   ```python
   class SLATrackingOperator(BaseOperator):
       def __init__(self, table: str, sla_hours: float, **kwargs):
           super().__init__(**kwargs)
           self.table = table
           self.sla_hours = sla_hours

       def execute(self, context):
           # Record SLA measurement
           execution_date = context['execution_date']
           actual_time = datetime.utcnow()
           expected_time = execution_date + timedelta(hours=self.sla_hours)
           met = actual_time <= expected_time
           # Insert into sla_history table
           self.log.info(f"SLA {'MET' if met else 'BREACHED'}: expected by {expected_time}, actual {actual_time}")
   ```

3. **Build an order event producer**:
   ```python
   from confluent_kafka import Producer
   import json

   class OrderEventProducer:
       def __init__(self, bootstrap_servers: str = 'localhost:9092'):
           self.producer = Producer({
               'bootstrap.servers': bootstrap_servers,
               'acks': 'all',  # wait for all replicas
               'retries': 3,
               'linger.ms': 10,  # batch messages for 10ms
               'batch.size': 16384,
           })

       def publish_order(self, order: dict):
           key = str(order['order_id']).encode('utf-8')
           value = json.dumps(order).encode('utf-8')

           self.producer.produce(
               topic='novamart.orders.created',
               key=key,  # partition by order_id
               value=value,
               callback=self._delivery_report
           )
           self.producer.poll(0)  # trigger callbacks

       def _delivery_report(self, err, msg):
           if err:
               print(f"Delivery failed: {err}")
           else:
               print(f"Delivered to {msg.topic()}[{msg.partition()}] @ offset {msg.offset()}")

       def flush(self):
           self.producer.flush()
   ```

4. **Build a consumer — analytics aggregator**:
   ```python
   from confluent_kafka import Consumer, KafkaError

   class OrderAnalyticsConsumer:
       def __init__(self, group_id: str = 'analytics-aggregator'):
           self.consumer = Consumer({
               'bootstrap.servers': 'localhost:9092',
               'group.id': group_id,
               'auto.offset.reset': 'earliest',
               'enable.auto.commit': False,  # manual commit for at-least-once
           })
           self.consumer.subscribe(['novamart.orders.created'])

       def process_messages(self):
           while True:
               msg = self.consumer.poll(timeout=1.0)
               if msg is None:
                   continue
               if msg.error():
                   if msg.error().code() == KafkaError._PARTITION_EOF:
                       continue
                   raise Exception(f"Consumer error: {msg.error()}")

               order = json.loads(msg.value().decode('utf-8'))
               self._process_order(order)

               # Manual commit after successful processing
               self.consumer.commit(message=msg)

       def _process_order(self, order: dict):
           # Update real-time aggregations
           print(f"Processing order {order['order_id']}: ${order['total_amount']}")
   ```

5. **Demonstrate consumer groups**:

### Common Mistakes

- Using Kafka as a database — Kafka is a log, not a queryable store (use it for transport, not storage)

## Task 107: Stream Processing Fundamentals

### Business Context
Raw CDC events and clickstream events flow through Kafka. But consumers need aggregated, enriched data — not raw events. You need a stream processing layer that: enriches order events with product data, aggregates clickstream into sessions, and computes real-time metrics. Ravi says: "Start with Flink SQL or Kafka Streams — pick one and build three real processing jobs."

### Objective
Build stream processing jobs that transform raw Kafka events into enriched, aggregated streams — using Flink SQL, Kafka Streams, or Spark Structured Streaming.

### Technical Requirements
- **Tools**: Flink SQL (recommended) or Spark Structured Streaming, Kafka
- **Skills**: Stream processing, stateful operations, windowed aggregations, stream-table joins

### Step-by-Step Guidance
1. **Option A: Flink SQL** (recommended for new stream processing):
   ```sql
   -- Create a Kafka source table
   CREATE TABLE order_events (
       order_id INT,
       user_id INT,
       total_amount DECIMAL(10,2),
       status STRING,
       event_time TIMESTAMP(3),
       WATERMARK FOR event_time AS event_time - INTERVAL '30' SECOND
   ) WITH (
       'connector' = 'kafka',
       'topic' = 'novamart.orders.created',
       'properties.bootstrap.servers' = 'localhost:9092',
       'format' = 'json',
       'scan.startup.mode' = 'earliest-offset'
   );

   -- Tumbling window aggregation: orders per minute
   SELECT
       TUMBLE_START(event_time, INTERVAL '1' MINUTE) AS window_start,
       COUNT(*) AS order_count,
       SUM(total_amount) AS revenue,
       COUNT(DISTINCT user_id) AS unique_customers
   FROM order_events
   GROUP BY TUMBLE(event_time, INTERVAL '1' MINUTE);
   ```

2. **Option B: Spark Structured Streaming**:
   ```python
   from pyspark.sql import SparkSession
   from pyspark.sql.functions import *

   spark = SparkSession.builder \
       .appName("NovaMart-Stream") \
       .getOrCreate()

   # Read from Kafka
   orders_stream = spark \
       .readStream \
       .format("kafka") \
       .option("kafka.bootstrap.servers", "localhost:9092") \
       .option("subscribe", "novamart.orders.created") \
       .option("startingOffsets", "earliest") \
       .load()

   # Parse JSON
   order_schema = StructType([
       StructField("order_id", IntegerType()),
       StructField("user_id", IntegerType()),
       StructField("total_amount", DoubleType()),
       StructField("event_time", TimestampType()),
   ])

   parsed = orders_stream \
       .select(from_json(col("value").cast("string"), order_schema).alias("data")) \
       .select("data.*")

   # Windowed aggregation
   windowed = parsed \
       .withWatermark("event_time", "30 seconds") \
       .groupBy(window("event_time", "1 minute")) \
       .agg(
           count("*").alias("order_count"),
           sum("total_amount").alias("revenue"),
           countDistinct("user_id").alias("unique_customers")
       )

   # Write to console (or Kafka, or database)
   query = windowed \
       .writeStream \
       .outputMode("update") \
       .format("console") \
       .option("truncate", False) \
       .start()

   query.awaitTermination()
   ```

3. **Build 3 processing jobs**:
   - **Job 1**: Real-time order metrics (orders/min, revenue/min) — tumbling window
   - **Job 2**: Clickstream sessionization — group events into sessions with 30-min gap detection
   - **Job 3**: Order enrichment — join order events with product dimension (stream-table join)

4. **Stream-table join** (enrich orders with product data):
   ```python
   # Spark: join stream with static (or slowly-changing) dimension
   products_df = spark.read.parquet("/data/dim_products/")  # static table

   enriched = parsed.join(
       broadcast(products_df),
       parsed.product_id == products_df.product_id,
       "left"
   )
   ```

5. **Write results to a sink**:
   ```python
   # Write to Kafka topic
   enriched.selectExpr("to_json(struct(*)) AS value") \
       .writeStream \
       .format("kafka") \
       .option("kafka.bootstrap.servers", "localhost:9092") \
       .option("topic", "novamart.orders.enriched") \
       .option("checkpointLocation", "/checkpoints/enriched") \
       .start()
   ```

### Expected Output
- Stream processing jobs: `tasks/107_stream_processing/jobs/`
- Docker Compose with Flink or Spark: `tasks/107_stream_processing/docker-compose.yaml`
- Demo: produce events → stream processing → aggregated output in real-time
- Concepts doc explaining watermarks, windows, and exactly-once in stream processing

### Common Mistakes
- No watermark — windows never close, memory grows forever
- Using processing time instead of event time — out-of-order events assigned to wrong windows
- Stream-stream join without bounds — both streams are kept in state forever (memory explosion)
- Not setting checkpoint location — no fault tolerance; crash = restart from beginning

Implement a sliding window in addition to tumbling, and compare the outputs.

1. **Design the Medallion layers**:

   ```
   │   ├── orders/
   │   │   ├── dt=2024-07-20/
   │   │   │   ├── orders_001.parquet
   │   │   │   └── orders_002.parquet
   │   ├── clickstream/
   │   └── vendor_inventory/
   ├── silver/                          # Cleaned, conformed, deduplicated
   │   ├── orders/                      # Delta/Iceberg tables
   │   ├── users/
   │   └── order_items/
   ├── gold/                            # Business-level aggregations
   │   ├── daily_revenue/
   │   ├── customer_segments/
   │   └── product_rankings/
   └── _metadata/
       ├── schemas/
       └── quality_reports/
   ```

2. **Bronze layer** — raw ingestion:
   ```python
   # Bronze: land raw data as-is, add metadata
   def write_bronze(df: DataFrame, table_name: str, source: str):
       bronze_df = df \
           .withColumn("_source", lit(source)) \
           .withColumn("_ingested_at", current_timestamp()) \
           .withColumn("_filename", input_file_name())

       bronze_df.write \
           .format("delta") \
           .mode("append") \
           .partitionBy("dt") \
           .save(f"s3://novamart-data-lake/bronze/{table_name}/")
   ```

3. **Silver layer** — cleaned and conformed:
   ```python
   # Silver: clean, deduplicate, enforce schema, SCD2
   def bronze_to_silver_orders(spark: SparkSession):
       bronze = spark.read.format("delta").load("s3://novamart-data-lake/bronze/orders/")

       silver = bronze \
           .dropDuplicates(["order_id"]) \
           .filter(col("order_id").isNotNull()) \
           .filter(col("total_amount") >= 0) \
           .withColumn("order_date", to_date(col("order_date"))) \
           .withColumn("total_amount", col("total_amount").cast("decimal(10,2)")) \
           .drop("_source", "_filename")

       # Merge into silver (upsert)
       silver_table = DeltaTable.forPath(spark, "s3://novamart-data-lake/silver/orders/")
       silver_table.alias("target") \
           .merge(silver.alias("source"), "target.order_id = source.order_id") \
           .whenMatchedUpdateAll() \
           .whenNotMatchedInsertAll() \
           .execute()
   ```

4. **Gold layer** — business aggregations:
   ```python
   # Gold: pre-computed aggregations for dashboards
   def silver_to_gold_daily_revenue(spark: SparkSession):
       orders = spark.read.format("delta").load("s3://novamart-data-lake/silver/orders/")
       items = spark.read.format("delta").load("s3://novamart-data-lake/silver/order_items/")

       daily = orders.join(items, "order_id") \
           .groupBy(to_date("order_date").alias("date")) \
           .agg(
               sum("line_total").alias("revenue"),
               countDistinct("order_id").alias("orders"),
               countDistinct("user_id").alias("customers"),
               avg("line_total").alias("avg_item_value")
           )

       daily.write \
           .format("delta") \
           .mode("overwrite") \
           .save("s3://novamart-data-lake/gold/daily_revenue/")
   ```

5. **Delta Lake features**:
   ```python
   # Time travel — query previous versions
   spark.read.format("delta").option("versionAsOf", 5).load(silver_path)

   # History
   delta_table = DeltaTable.forPath(spark, silver_path)
   delta_table.history().show()

   # Schema enforcement — reject writes with wrong schema
   # Schema evolution — allow safe additions
   silver_df.write.format("delta") \
       .option("mergeSchema", "true") \
       .mode("append") \
       .save(silver_path)
   ```

6. **Quality gates between layers**:
   - Bronze → Silver: schema validation, null checks, deduplication
   - Silver → Gold: business rule validation, referential integrity, freshness

### Expected Output
- Layer implementations: `tasks/108_medallion/bronze/`, `silver/`, `gold/`
- Pipeline: `tasks/108_medallion/pipeline.py`
- Architecture diagram: `tasks/108_medallion/architecture.md`
- Demo: raw files → bronze → silver → gold with quality gates at each step

### Common Mistakes
- Business logic in Bronze — Bronze should be a faithful copy of the source
- Skipping Silver — going straight from raw to aggregated loses the clean intermediate layer
- No quality gates between layers — bad data propagates from Bronze to Gold
- Not using a table format (Delta/Iceberg) — raw Parquet has no ACID, no schema enforcement, no time travel

   ```python
       def push_features(self, feature_group: str, df: pd.DataFrame):
           """Push batch-computed features to Redis for online serving"""
           for _, row in df.iterrows():
               key = f"features:{feature_group}:{row['user_id']}"
   ```

- **Skills**: Event sourcing, CQRS, event stores, projections, event replay

### Step-by-Step Guidance
1. **Design the event store**:
   ```sql

       event_id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
       aggregate_id INTEGER NOT NULL,  -- order_id
       event_type VARCHAR(100) NOT NULL,
       event_data JSONB NOT NULL,
       event_version INTEGER NOT NULL,
       created_at TIMESTAMP DEFAULT NOW(),
       created_by VARCHAR(100),
       UNIQUE(aggregate_id, event_version)  -- prevent duplicate events
   );

   CREATE INDEX idx_order_events_aggregate ON order_events(aggregate_id, event_version);
   ```

2. **Define event types**:
   ```python
   # Events are immutable facts
   @dataclass(frozen=True)
   class OrderCreated:
       order_id: int
       user_id: int
       items: list
       total_amount: float
       timestamp: datetime

   @dataclass(frozen=True)
   class OrderConfirmed:
       order_id: int
       confirmed_by: str
       timestamp: datetime

   @dataclass(frozen=True)
   class OrderShipped:
       order_id: int
       tracking_number: str
       carrier: str
       timestamp: datetime

   @dataclass(frozen=True)
   class OrderDelivered:
       order_id: int
       delivered_at: datetime
       signature: str

   @dataclass(frozen=True)
   class OrderCancelled:
       order_id: int
       reason: str
       refund_amount: float
       timestamp: datetime
   ```

3. **Build the event store**:
   ```python
   class EventStore:
       def append(self, aggregate_id: int, event: Event, expected_version: int):
           """Append event with optimistic concurrency check"""
           current_version = self._get_latest_version(aggregate_id)
           if current_version != expected_version:
               raise ConcurrencyError(f"Expected version {expected_version}, got {current_version}")
           self._insert_event(aggregate_id, event, expected_version + 1)

       def get_events(self, aggregate_id: int) -> List[Event]:
           """Get all events for an order, in order"""
           rows = query("SELECT * FROM order_events WHERE aggregate_id = %s ORDER BY event_version", aggregate_id)
           return [deserialize_event(row) for row in rows]

       def replay(self, aggregate_id: int) -> Order:
           """Rebuild current state by replaying all events"""
           events = self.get_events(aggregate_id)
           order = Order()
           for event in events:
               order.apply(event)
           return order
   ```

4. **Build CQRS read models (projections)**:
   ```python
   class OrderSummaryProjection:
       """Materialized view optimized for querying current order state"""
       def project(self, event: Event):
           if isinstance(event, OrderCreated):
               self._insert_summary(event)
           elif isinstance(event, OrderShipped):
               self._update_summary(event.order_id, status='shipped',
                                     tracking_number=event.tracking_number)
           elif isinstance(event, OrderDelivered):
               self._update_summary(event.order_id, status='delivered',
                                     delivered_at=event.delivered_at)

   class OrderTimelineProjection:
       """Shows all state transitions for an order"""
       def project(self, event: Event):
           self._insert_timeline(event.order_id, event.__class__.__name__,
                                  event.timestamp, event)
   ```

5. **Query patterns**:
   ```sql

   -- Event history: full timeline
   SELECT event_type, event_data, created_at
   FROM order_events
   WHERE aggregate_id = 10001
   ORDER BY event_version;

   -- Analytics: average time from confirmed to shipped

   FROM order_events confirmed
   JOIN order_events shipped
     ON confirmed.aggregate_id = shipped.aggregate_id
   WHERE confirmed.event_type = 'OrderConfirmed'
     AND shipped.event_type = 'OrderShipped';
   ```

6. **Event replay**: rebuild all read models from scratch by replaying all events

### Step-by-Step Guidance
1. **Audit current costs** using Snowflake ACCOUNT_USAGE:
   ```sql
   -- Credit usage by warehouse (last 30 days)
   SELECT
       warehouse_name,
       SUM(credits_used) AS total_credits,
       ROUND(SUM(credits_used) * 3.00, 2) AS estimated_cost,  -- $3/credit for enterprise
       COUNT(DISTINCT DATE(start_time)) AS active_days,
       ROUND(SUM(credits_used) / COUNT(DISTINCT DATE(start_time)), 2) AS credits_per_day
   FROM SNOWFLAKE.ACCOUNT_USAGE.WAREHOUSE_METERING_HISTORY
   WHERE start_time >= DATEADD(day, -30, CURRENT_DATE)
   GROUP BY warehouse_name
   ORDER BY total_credits DESC;

   -- Most expensive queries
   SELECT
       query_id,
       query_text,
       warehouse_name,
       execution_time / 1000 AS seconds,
       credits_used_cloud_services,
       partitions_scanned,
       bytes_scanned / (1024*1024*1024) AS gb_scanned
   FROM SNOWFLAKE.ACCOUNT_USAGE.QUERY_HISTORY
   WHERE start_time >= DATEADD(day, -7, CURRENT_DATE)
   ORDER BY execution_time DESC
   LIMIT 50;

   -- Storage costs
   SELECT
       TABLE_CATALOG,
       TABLE_SCHEMA,
       TABLE_NAME,
       ROUND(ACTIVE_BYTES / (1024*1024*1024), 2) AS active_gb,
       ROUND(TIME_TRAVEL_BYTES / (1024*1024*1024), 2) AS time_travel_gb,
       ROUND(FAILSAFE_BYTES / (1024*1024*1024), 2) AS failsafe_gb,
       ROUND((ACTIVE_BYTES + TIME_TRAVEL_BYTES + FAILSAFE_BYTES) / (1024*1024*1024), 2) AS total_gb
   FROM SNOWFLAKE.ACCOUNT_USAGE.TABLE_STORAGE_METRICS
   WHERE TABLE_CATALOG = 'NOVAMART_DW'
   ORDER BY total_gb DESC
   LIMIT 20;
   ```
