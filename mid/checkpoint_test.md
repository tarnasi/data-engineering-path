# Mid Checkpoint Test

Take this test around the middle of the level.

## Test Goal

Check whether you can move from building scripts to designing dependable systems.

## Tasks

1. Create a dbt model flow with staging and mart layers.
2. Add schema tests and one custom test.
3. Build one Airflow DAG with retries and a clear failure path.
4. Design one star schema and define its grain.
5. Implement or explain one historical dimension pattern.
6. Improve one slow query and explain why the change helps.

## Pass Standard

You pass if you can:

- explain the architecture clearly
- justify one tradeoff in each design
- show tests or validation protecting the output
- reason about reruns and backfills safely

## If You Fail

Spend 2 to 3 more weeks on:

- dbt layering
- Airflow dependency design
- dimensional modeling
- SQL optimization
