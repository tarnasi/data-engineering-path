# Mid Mastery Roadmap

## Purpose

This level turns you from a capable builder into a system designer.
You should be able to create workflows that other people can trust and reuse.

## Time

- Target: 24 to 32 weeks
- Slow path: 36 to 40 weeks
- Daily work: 2.5 to 3.5 hours ideal

## Scope

Focus on:

- advanced SQL and warehouse thinking
- dbt models, tests, and snapshots
- Airflow DAGs with failure handling
- stronger Python structure and tests
- historical modeling and dimensional design
- performance thinking and safe backfills

## Task Mapping In This Repository

Use these as Mid preparation first:

- Junior Task 11 to Junior Task 20

Then continue with:

- Mid task files in order

## Weekly Structure

- Day 1: warehouse SQL and modeling
- Day 2: dbt and transformations
- Day 3: Airflow and orchestration
- Day 4: Python testing and refactoring
- Day 5: reliability, backfills, or debugging
- Day 6: design review and writing
- Day 7: rest or light review

## Mastery Targets

You should be able to:

- design staging to mart flows cleanly
- implement tests around transformations
- build Airflow DAGs with retries and clear failure paths
- design star schemas and historical dimensions
- improve slow queries with evidence
- document technical tradeoffs clearly

## Exit Condition

Do not leave this level until you can design and run a small analytics platform with testing, documentation, and safe reruns.
