# Beginner Homework Guide

## How To Do The Homework

For every task at this level:

1. Read the task once without coding.
2. Write down the input files, target output, and business question.
3. Solve the smallest working version first.
4. Rerun the solution from the beginning.
5. Clean the code only after it works.
6. Save short notes about what confused you.

## What You Must Explain After Each Task

After each task, explain these points in simple language:

- What problem this task solves
- What data came in
- What data went out
- What transformation happened
- What errors or data problems you found
- How you checked the result was correct
- What you would improve next time

## Beginner Quality Rules

- Do not hide bad rows silently.
- Do not hardcode too many paths in many places.
- Do not move to harder tools when SQL or Python basics still feel unstable.
- Keep scripts short and readable.
- Prefer one clear solution over one clever solution.

## Submission Standard

A task is complete only when you have:

- working code
- output file or database result
- short written explanation
- one rerun without manual fixing
